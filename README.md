# UMIRHAK 2026 Frontend

Фронтенд-часть проекта «Честный Диплом» для работы с реестром дипломов.

## Что это за проект

«Честный Диплом» - это веб-платформа для проверки подлинности дипломов через единый реестр.

Проект решает задачу доверенной и быстрой верификации диплома между тремя основными сторонами:

- ВУЗ публикует и сопровождает записи о выпускниках.
- Выпускник управляет доступом к своей записи и делится проверкой через QR/ссылку.
- Работодатель (HR) проверяет диплом в несколько кликов, без долгой переписки с учебным заведением.

Ключевая цель: сократить время проверки, снизить риск поддельных документов и сделать процесс прозрачным для всех ролей.

## Основные возможности фронтенда

- Ролевая навигация и отдельные кабинеты для: ВУЗа, студента, HR, администратора и суперпользователя.
- Публичная страница и сценарий проверки диплома по ссылке/QR.
- Формы входа, восстановления доступа и регистрации по роли.
- Интеграция с REST API бэкенда (Ktor) через единый слой API-клиентов.
- Демонстрационный режим с тестовыми аккаунтами.

## Как запустить проект

### Требования

- Node.js 20+ (рекомендуется LTS)
- npm 10+
- Запущенный backend API (по умолчанию на http://localhost:8080)

### Установка зависимостей

```bash
cd umirhak2026_front
npm install
```

### Запуск в режиме разработки

```bash
npm run dev
```

По умолчанию Vite поднимает фронтенд на http://localhost:5173.

### Сборка production

```bash
npm run build
```

### Локальный preview production-сборки

```bash
npm run preview
```

### Настройка адреса API

Фронтенд использует переменную окружения VITE_API_BASE_URL.

Если переменная не задана, используется fallback: протокол + hostname текущего хоста и порт 8080.

Пример:

```bash
# .env.local
VITE_API_BASE_URL=http://localhost:8080
```

## Структура проекта

Ключевые директории:

- src/api - API-клиенты и общая конфигурация запросов к бэкенду.
- src/auth - контекст авторизации, роли, демо-аккаунты и профили сессии.
- src/components - переиспользуемые UI-компоненты и оболочки страниц.
- src/pages - страницы приложения (включая кабинеты по ролям).
- src/hooks - кастомные React hooks.
- src/utils - утилиты (например, импорт/парсинг дипломов).
- src/styles и src/auth.css - глобальные/тематические стили.
- demo-keys - демо CSV-файлы для тестов импорта.

## Как пользоваться решением

### Сценарий 1. Вход под нужной ролью

1. Откройте страницу входа.
2. Выберите роль или перейдите по нужному route (например, /login, /login/vuz, /login/admin).
3. Используйте демо-логин и пароль из раздела выше.
4. После входа откроется ролевой кабинет.

### Сценарий 2. Проверка диплома работодателем

1. Войдите как HR.
2. Перейдите в кабинет работодателя.
3. Найдите запись диплома через форму поиска/проверки.
4. Получите итоговый статус верификации.

### Сценарий 3. Работа студента с QR

1. Войдите как студент.
2. Подтвердите данные диплома.
3. Сгенерируйте QR/ссылку для проверки.
4. Передайте ссылку работодателю и при необходимости отзовите доступ.

### Сценарий 4. Работа ВУЗа с реестром

1. Войдите как представитель ВУЗа.
2. Добавьте диплом вручную или импортируйте CSV.
3. Проверьте, что запись появилась в реестре.
4. При необходимости аннулируйте запись по номеру.
